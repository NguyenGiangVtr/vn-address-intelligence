Bước 1: Thiết lập Cohort thử nghiệm

Lấy mẫu dữ liệu từ Ground Truth và áp dụng các mô hình nhiễu thực tế.

📥 Input: Dữ liệu từ bảng prq.ground_truth (địa chỉ đã được kiểm chứng)
📤 Output: Bộ dữ liệu thử nghiệm với nhiễu được lưu vào prq.supa_benchmark_specimen
Thông tin về lần thử sẽ hiện ở đây.
⚡ Muốn chạy lại với Luồng Tự động?
Nếu bạn muốn chạy lại bộ thử này với pipeline tự động (không cần CSV thủ công):
  1. Mở phần "Cấu hình bộ thử mới" bên dưới
  2. Tích checkbox "TỰ ĐỘNG CHẠY NORMALIZATION"
  3. Chọn cấu hình model mong muốn
  4. Click "Bắt đầu Workflow"
Cấu hình bộ thử mới
Số lượng địa chỉ tối đa muốn lấy từ Ground Truth.
Dùng cùng một số seed để tái lập chính xác bộ dữ liệu này sau này.
Quy định cách thức giả lập lỗi gõ của người dùng.
Để trống để tự động tạo theo pattern: 1000-42-SUP-1.0.0
Nếu bật, hệ thống sẽ tự động chạy pipeline AI (NER + Retrieval + LLM) ngay sau khi trích mẫu, không cần import CSV thủ công. Phù hợp cho ablation study và so sánh các cấu hình model.

Bước 2: Xem trước Dữ liệu mẫu

Kiểm tra dữ liệu đã được trích xuất và áp dụng nhiễu.

📥 Input: Dữ liệu từ Bước 1 (bộ thử đã được tạo)
📤 Output: File CSV có thể tải xuống để chạy pipeline chuẩn hóa bên ngoài, hoặc tiếp tục sang Bước 3 để chuẩn hóa
🎯

Bạn có 2 lựa chọn để tiếp tục:

🔄 Luồng Ngoại vi (Manual)
  1. Click nút "Tải CSV" bên dưới
  2. Chạy pipeline của bạn (Python, API, etc.)
  3. Điền kết quả vào cột pred_standardized
  4. Sang Bước 3 → Upload file CSV đã điền
⚡ Luồng Tự động (Auto)
  1. Quay lại Bước 1
  2. Tích checkbox "TỰ ĐỘNG CHẠY NORMALIZATION"
  3. Chọn cấu hình model (NER, Retrieval, LLM)
  4. Click "Bắt đầu Workflow"
  5. Hệ thống tự động chạy và chuyển sang Bước 4
💡 Gợi ý: Nếu chỉ muốn xem demo nhanh, dùng Luồng Tự động. Nếu muốn test model riêng, dùng Luồng Ngoại vi.

Bảng đối soát dữ liệu mẫu

ID Tham chiếu (V2) Dữ liệu nhiễu (Input) Dự đoán (Output) Tầng Trạng thái
Vui lòng chọn hoặc tạo bộ thử ở Bước 1.
Hiển thị 0 địa chỉ

Bước 3: Chuẩn hóa Địa chỉ

Chạy pipeline AI để chuẩn hóa các địa chỉ có nhiễu.

📥 Input: Dữ liệu nhiễu từ Bước 2 (cột noisy_raw_address)
📤 Output: Địa chỉ đã chuẩn hóa (cột pred_standardized) được lưu vào DB để đánh giá ở Bước 4
Phương án 1: Nhập kết quả từ Pipeline ngoại vi
📋 Quy trình:
1️⃣ Tải file CSV từ Bước 2 (nút "Tải CSV cho Pipeline ngoại vi")
2️⃣ Chạy pipeline chuẩn hóa của bạn (Python script, API riêng, v.v.)
3️⃣ Điền kết quả vào cột pred_standardized
4️⃣ Upload file CSV đã điền kết quả ở form bên dưới
File phải có cột 'specimen_id' và 'pred_standardized' đã được điền.
Thông tin này sẽ được lưu để đối soát hiệu năng giữa các phiên bản model.
Kiểm tra định dạng file mà không lưu vào database. Dùng để validate trước khi import thật.
Không đo thời gian import. Chỉ bật nếu file CSV của bạn đã có cột latency_ms từ pipeline.
🧪 Chế độ Demo (Không có file CSV?)

Nếu bạn chưa có file CSV kết quả, có thể dùng chế độ demo để test flow. Hệ thống sẽ tự động copy dữ liệu tham chiếu V2 làm kết quả (EM = 100%).

Phương án 2: Tự động chạy Pipeline trên Server
⚡ Chế độ Tự động (Ablation Study)

Pipeline AI sẽ tự động chạy trên server ngay sau khi tạo bộ thử ở Bước 1.

🔧 Cách kích hoạt:
1️⃣ Quay lại Bước 1
2️⃣ Tích vào checkbox "TỰ ĐỘNG CHẠY NORMALIZATION"
3️⃣ Chọn các thành phần model (NER, Retrieval, LLM)
4️⃣ Click "Bắt đầu Workflow"
5️⃣ Hệ thống sẽ tự động: Extract → Normalize → Eval
✅ Ưu điểm của Phương án 2:
  • Không cần tải file CSV thủ công
  • Không cần chạy pipeline riêng
  • Tự động đo latency chính xác
  • Phù hợp cho ablation study (so sánh các cấu hình model)
Nhật ký hệ thống chi tiết
